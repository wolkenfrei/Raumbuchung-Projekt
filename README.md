## 🎯 Projektidee  
Eine einfache Streamlit-App zur Buchung von Räumen und Plätzen.  

Grundidee:  
Eine Streamlit-App zur Buchung von Räumen und Plätzen  
mit dem Ziel, Nutzung, Zustand und Auffälligkeiten nachvollziehbar zu machen.

---

## 💡 Warum dieses Projekt  
Ich mache das Projekt zum Spaß und um meine Kenntnisse zu erweitern.  

Ziel ist es dabei:

- Python besser zu verstehen  
- mit Streamlit zu arbeiten  
- einfache Logik sauber umzusetzen  
- Schritt für Schritt ein eigenes System aufzubauen  

---

## ⚙️ Warum Streamlit  
Streamlit ist für so ein Projekt einfach praktisch.

- läuft direkt in Python  
- schnell Ergebnisse sichtbar  
- kein HTML / CSS notwendig  
- gut für kleine Tools und Lernprojekte  

---

## 🧩 Aktueller Stand  

Die App kann aktuell:

- Räume anzeigen  
- Plätze anzeigen  
- Platz auswählen  
- Platz buchen  
- erkennen, ob ein Platz schon belegt ist  

---

## 🔄 Geplante Logik (so soll es später laufen)

- User wählt einen Raum  
- User wählt einen freien Platz  
- User bucht den Platz  
- User arbeitet dort  
- User hat Feierabend und gibt den Platz wieder frei  

Beim Freigeben soll es dann geben:

- Möglichkeit zur Bewertung  
- optionales Textfeld (z. B. „Tisch dreckig“, „Technik defekt“)  

---

## 🧾 Weitere Ideen

- Eingabe eines Usernamens bei der Buchung  
- Speicherung der Buchungen (wer war wo?) - Nicht sichtbar für andere Personen!
- später Datenbank-Anbindung  

Ziel dabei:

- nachvollziehen können, wer welchen Platz genutzt hat  
- Probleme erkennen (z. B. Räume oft dreckig oder Technik kaputt)  

---

## 📊 Admin / Auswertung (später)

- Übersicht über alle Buchungen  
- Tagesübersicht  
- erkennen, welche Räume Probleme machen  
- z. B.:
  - Raum immer dreckig  
  - Technik funktioniert öfter nicht  

---

## 🏗 Projektstruktur  

Der Code ist in Blöcke unterteilt:
